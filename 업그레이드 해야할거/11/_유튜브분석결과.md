## 바로 쓸 수 있는 것들 (쓸만함)

### (4) FD 에이전시 "델타봇" — 슬랙 기반 AI 워크플로우
- 에이전트 단일 책임 원칙, 피드백 루프로 토론 대체
- AI가 인간에게 태스크를 뿌리는 역전 구조
- **적용**: 시은(오케스트레이터)을 델타봇처럼 "놓친 사항 지적 + 재호출"까지 확장

### (5) Cokkak DIR — 텔레그램 멀티 AI 에이전트 팀
- 텔레그램 그룹에 에이전트 봇 여러 개 → 자율 역할 분담/토론
- 스마트폰에서 아이디어 한 줄 → 에이전트 팀이 알아서 실행
- **적용**: "에이전트끼리 메시지 주고받으며 토론하는 구조" 핵심 개념

### (6) 하네스 엔지니어링 — 멱등성 있는 AI 코드 생성
- 설계문서+린터+평가로 어떤 AI 모델이든 동일한 결과물 보장
- CPS(Context-Problem-Solution) 프레임워크, 이밸류에이션 프레임
- **적용**: 종범 설계서에 CPS+린터 규칙 포함, Wave 4 QA에 린터 훅 추가

### (8) Google Workspace CLI — AI용 CLI 설계
- AI 에이전트가 쓰도록 설계된 CLI (Gmail, Drive, Sheets 조작)
- JSON 입출력, 레시피 패턴, 드라이런 기능
- **적용**: 에이전트 간 통신 JSON 표준화, 출시 후 모니터링에 Workspace 연동

### (10) OpenClaw 16개 에이전트 — 18세 Vim 사례
- Cron Job으로 24/7 자율 실행 (밤새 코드리뷰, 2시간마다 버그 모니터링)
- 에이전트 간 체인, Mission Control 대시보드
- **적용**: Wave 7 모니터링 Cron 추가, Mission Control UI 구축

### (12) Claude Code Channels — 외부→클로드 푸시
- 외부 서비스(텔레그램/디스코드)에서 클로드코드 세션에 프롬프트 push
- MCP 역방향 통신
- **적용**: 리안이 텔레그램에서 아이디어 던지면 자동 실행, 에러 발생 시 자동 수정 트리거
