# 06. Pain Point 수집기 (Indie Hacker 기회 탐지 시스템)

> 아직 아무도 안 만든 진짜 문제를 찾아오는 자동화 시스템

---

## 한 줄 요약
커뮤니티에서 "이거 왜 없냐", "이거 불편해" 글을 24시간 수집해서
"지금 만들면 바로 팔리는 문제 TOP 5"를 매주 리포트해주는 B2B SaaS.

---

## 누가 쓰나

| 고객 | 이유 |
|------|------|
| 1인 개발자 / 인디해커 | 아이디어 발굴에 시간 너무 많이 씀 |
| 스타트업 기획팀 | 시장 검증 전에 진짜 페인포인트 확인 필요 |
| VC / 액셀러레이터 | 포트폴리오 기업 아이디어 검증 |
| 리안 같은 AI CEO | 자동으로 다음 프로젝트 후보 확보 |

---

## 뭘 수집하나

### 소스
- Reddit (r/entrepreneur, r/smallbusiness, r/startups, r/sidehustle 등)
- Hacker News (Ask HN: "does anyone else hate X?")
- Product Hunt 댓글 ("wish this had X feature")
- Twitter/X (#buildinpublic, #indiemaker)
- 국내: 블라인드, 클리앙, 네이버카페, 에브리타임

### 수집 패턴 (키워드 클러스터)
```
"왜 없냐" / "why isn't there"
"이거 해결해주는" / "tool that does X"
"매번 수동으로" / "I do this manually every"
"이거 진짜 불편" / "this is so annoying"
"누가 만들어줬으면" / "someone should make"
"X 쓰는데 Y가 없어서" / "X but missing Y"
```

---

## 어떻게 돌아가나

```
[백그라운드 스케줄러 — 6시간마다]
    ↓
소스별 크롤러 + API 수집
    ↓
GPT로 "진짜 페인포인트인지" 분류
    ↓
유사 문제 클러스터링 (같은 문제 여러 곳에서 나왔는지)
    ↓
점수 산정: 언급 수 × 감정 강도 × 검색량 × 경쟁 앱 유무
    ↓
주간 리포트 생성
    ↓
대시보드 표시 + 이메일/슬랙 알림
```

### 리포트 예시
```
📌 이번 주 TOP Pain Point #1

문제: "세금계산서 발행 후 수정 요청이 오면 매번 수작업"
언급: 47건 (Reddit 12, 블라인드 23, 네이버카페 12)
감정 강도: 매우 높음 (짜증/시간낭비 키워드 다수)
기존 해결책: 없음 (경쟁 앱 0개)
예상 고객: 중소기업 경리 담당자
만들면 팔릴 것 같은 이유: 반복 언급, 기존 솔루션 전무
```

## 왜 지금인가
- 인디해커 트렌드 급성장 (월 $10k MRR 달성 사례 매주 나옴)
- AI로 분류/요약 비용 극히 저렴해짐
- 아이디어 발굴 = 가장 큰 병목 → 돈 낼 의사 높음
- 국내는 아직 아무도 안 함
