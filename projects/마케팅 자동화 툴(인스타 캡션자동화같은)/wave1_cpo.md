# CPO 분석 - 소상공인 인스타그램 자동화 서비스 (instauto)

## 제품 전략
**"손 안 대고 인스타 운영하는 카페 사장님의 마케팅 비서"**

소상공인이 매일 반복하는 인스타그램 콘텐츠 제작 고통을 AI로 완전 자동화.
가입 후 3분 온보딩만 하면 매일 아침 '오늘의 게시물'이 만들어져 있고, 버튼 하나로 발행 완료.

---

## 타겟 + Pain

### 타겟
**1인 또는 소규모 카페/네일샵/식당 사장님 (20~40대)**
- 인스타그램이 중요하다는 건 알지만 시간이 없음
- 직원에게 맡기기엔 일관성 부족, 외주 맡기기엔 비용 부담
- 폰으로 대부분의 업무 처리 (PC 작업 기피)
- 월 2~3만원 수준의 마케팅 비용은 기꺼이 지불 의사 있음

### Pain
1. **시간 부족**: 영업 준비/마감 사이에 인스타 할 시간이 없음
2. **콘텐츠 막막함**: "오늘 뭘 올리지?" 매일 고민
3. **사진/문구 퀄리티**: 직접 만들면 어설프고, 외주는 비쌈
4. **일관성 유지 실패**: 2주 열심히 하다가 3개월 방치 반복

### 아하 모먼트
**"오전에 가게 오픈하면서 앱 열었는데, 이미 오늘 올릴 게시물이 만들어져 있네?"**
- 내 가게 분위기에 맞는 이미지와 캡션이 준비되어 있음
- '발행' 버튼 한 번 누르면 끝
- "이거 괜찮네?" 라고 느끼는 순간 = 전환 포인트

## MVP 범위 (Must Have)

### 1. 온보딩 플로우 (5분 이내 완료)
- 이메일/구글 가입
- 가게 정보 입력: 가게명, 업종(MVP는 카페 고정), 분위기 키워드 3개, 대표 메뉴 3개
- Instagram 비즈니스 계정 OAuth 연동

### 2. 매일 자동 콘텐츠 생성
- 매일 오전 8시 Cron Job으로 GPT-4o 캡션 + DALL-E 3 이미지 자동 생성
- 가게 정보 기반 개인화된 콘텐츠

### 3. 오늘의 콘텐츠 대시보드
- 생성된 이미지 + 캡션 미리보기
- 캡션 직접 수정 가능
- 재생성 버튼 (하루 3회 제한)

### 4. 발행 기능
- '지금 발행' 버튼 -> Instagram 즉시 업로드
- 예약 발행 (시간 선택)

### 5. 무료체험 -> 유료 전환
- 30일 무료 (카드 등록 없이)
- 만료 3일/1일 전 알림 이메일 (Resend)
- 토스페이먼츠 결제 연동

## MVP 제외 (나중에)

| 기능 | 제외 이유 |
|------|----------|
| 카페 외 업종 (네일샵, 식당) | 캡션/이미지 프롬프트 업종별 최적화 필요. MVP는 카페 하나에 집중 |
| 릴스/스토리 자동화 | 피드 이미지 1장이 핵심 가치 검증의 최소 단위 |
| 인사이트 분석 대시보드 | 발행 후 분석은 Phase 2. 먼저 "발행"까지 가는 게 급함 |
| 팀 멤버 초대/협업 | 1인 사장님 타겟. 다중 사용자는 PMF 확인 후 |
| 모바일 앱 (iOS/Android) | 반응형 웹으로 MVP 검증 -> 네이티브 앱은 Phase 2 |
| 해시태그 자동 추천 | GPT가 캡션에 해시태그 포함하도록 프롬프트로 해결 |
| 여러 게시물 동시 생성 | 하루 1개면 충분. 복잡도만 증가 |
| A/B 테스트 기능 | PMF 전에 과잉 |

## 성공 기준 (KPI)

### 30일 목표 (런칭 후)
| 지표 | 목표치 | 측정 방법 |
|------|--------|----------|
| 가입자 수 | 200명 | Supabase Auth 신규 가입 |
| 온보딩 완료율 | 60% 이상 | 가입 -> Instagram 연동 완료 비율 |
| 첫 발행 전환율 | 40% 이상 | 온보딩 완료자 중 1회 이상 발행 |
| 무료체험 -> 유료 전환율 | 10% 이상 | 30일 무료체험 종료자 중 결제 완료 |
| D7 리텐션 | 30% 이상 | 가입 7일 후 재방문 |

### 핵심 지표 (North Star Metric)
**"주간 활성 발행자 수 (WAP: Weekly Active Publishers)"**
- 정의: 최근 7일간 1회 이상 Instagram 발행한 유저 수
- 이유: 발행까지 가야 가치를 느낌. 생성만 하고 발행 안 하면 실패

## 배포 블로커

### 1. Meta Graph API 앱 심사 (Critical)
| 블로커 | 심각도 | 해결 방법 |
|--------|--------|----------|
| Instagram Basic Display -> Graph API 전환 | High | 비즈니스 계정 전용 Graph API 사용. 앱 심사 2~4주 소요 |
| 앱 심사 리젝 가능성 | High | 심사 전 테스트 유저로 개발 모드 사용. 심사 대비 Privacy Policy, TOS 페이지 필수 |

**대응**: 개발 모드로 MVP 완성 -> 테스트 유저 5명으로 검증 -> 심사 제출

### 2. OpenAI API 비용 통제
| 블로커 | 심각도 | 해결 방법 |
|--------|--------|----------|
| DALL-E 3 이미지당 $0.04~0.08 | Medium | Standard 1024x1024 사용 ($0.04). 재생성 하루 3회 제한 |
| GPT-4o 토큰 비용 | Low | 캡션 생성은 짧음. 토큰 제한 500 이내 |

**예상 비용**: 유저당 월 $1.5~2 (일 1회 생성, 재생성 평균 1회 가정)
**마진**: 월 19,900원 구독 시 약 70% 마진 (환율 1,350원 기준)

### 3. 토스페이먼츠 연동
| 블로커 | 심각도 | 해결 방법 |
|--------|--------|----------|
| 정기결제(빌링키) 심사 | Medium | 사업자등록 필요. 개인사업자도 가능 |

### 4. Vercel 배포 환경
| 블로커 | 심각도 | 해결 방법 |
|--------|--------|----------|
| Cron Job 실행 보장 | Low | Vercel Pro 플랜($20/월)으로 안정적 Cron 보장 |
| 서버리스 타임아웃 (10초) | Medium | DALL-E 생성 시간 5~15초. Pro 플랜 60초 타임아웃 |

## CTO에게 요청

### 기술 확인 필요 사항

1. **Meta Graph API 제약 확인**
   - Instagram 비즈니스 계정만 지원되는지 확인 (개인 계정 불가 가능성)
   - OAuth 토큰 갱신 주기 및 자동 갱신 구현 방안
   - 이미지 발행 시 URL 호스팅 필요 여부 (Cloudinary/Supabase Storage)

2. **DALL-E 3 이미지 품질/속도 테스트**
   - 카페 분위기 이미지 생성 품질 실제 테스트
   - 생성 시간 측정 (Vercel 서버리스 타임아웃 대응)

3. **Vercel Cron 안정성**
   - 매일 오전 8시 수백 명 동시 콘텐츠 생성 시 부하 테스트
   - 배치 처리 vs 큐(Queue) 도입 필요성

4. **Supabase Row Level Security**
   - 멀티테넌트 환경에서 유저 간 데이터 격리 구현 방안

5. **토스페이먼츠 정기결제**
   - 빌링키 발급 테스트 환경 확인
   - 결제 실패 시 재시도 로직

## 경쟁 우위

1. **완전 자동화**: 매일 알아서 만들어져 있음. 사장님은 "발행" 버튼만 누름
2. **한국 소상공인 특화**: 토스페이먼츠 결제, 한국어 캡션, 국내 카페 분위기

4. **무료체험 30일**: 카드 등록 없이 충분히 써보고 결정
