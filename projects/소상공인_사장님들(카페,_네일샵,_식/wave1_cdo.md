# CDO 분석 — 소상공인 인스타그램 자동화 서비스 (instauto)

---

## 디자인 비전

"매일 아침, 당신의 가게 이야기를 알아서 만들어드립니다"

따뜻하고 신뢰감 있는 디지털 어시스턴트. 기술을 몰라도 직관적으로 사용할 수 있으며, AI가 만든 결과물에 대한 통제감을 제공합니다. 화려한 대시보드보다 "오늘 뭘 올릴지 걱정하지 않아도 된다"는 안심감을 주는 것이 핵심입니다.

**톤 앤 매너**: 친근한 동네 친구, 프로페셔널하지만 격식 없는, 실용적이고 따뜻한

---

## UX 원칙

### 1. 0-클릭 가치 제공
- 로그인하자마자 오늘의 콘텐츠가 이미 생성되어 있어야 함
- 마음에 들면 버튼 하나로 발행 가능
- 설정은 나중에 해도 됨 (Progressive Disclosure)

### 2. 실수 방지 > 실수 복구
- 발행 전 반드시 미리보기 제공
- 즉시 발행 vs 예약 발행을 명확히 구분
- 재생성 횟수를 시각적으로 표시하여 무분별한 재생성 방지
- 인스타 연동 끊김 시 대시보드 상단에 명확한 알림

### 3. 모바일이 메인, 데스크톱은 보너스
- 모든 주요 액션은 엄지손가락 하나로 완료 가능
- 중요 버튼은 하단 48px 이상, 여백 충분히
- 긴 폼은 여러 단계로 분할 (온보딩: 가게 정보 → 인스타 연동)
- 스크롤 최소화: 한 화면에 핵심 정보 집중

## 핵심 화면 흐름

```
[랜딩 페이지]
   ↓ (30초 안에 가치 전달)
[회원가입/로그인] (Google 소셜 로그인 우선)
   ↓
[온보딩 1/2: 가게 정보]
   - 가게 이름
   - 분위기 키워드 3개 선택 (버튼 형태)
   - 대표 메뉴 3개 입력
   ↓
[온보딩 2/2: 인스타 연동]
   - Instagram 계정 연결 (OAuth)
   - "연동하면 뭐가 좋은지" 설명
   ↓
[대시보드/홈 - 오늘의 콘텐츠]
   - 생성된 이미지 + 캡션 카드
   - 수정 (캡션 편집 / 이미지 재생성)
   - 즉시 발행 or 예약 발행
   ↓ (좌측 사이드바/하단 탭바)
[발행 히스토리]
   - 지난 콘텐츠 목록
   - 성과 간단 표시 (발행 여부, 예약 상태)
   ↓ (설정 아이콘)
[설정]
   - 가게 정보 수정
   - 인스타 연동 관리
   - 구독 플랜 / 결제 정보
```

---

## 각 화면 핵심 요소

### 1. 랜딩 페이지
**목표**: 30초 안에 "AI가 매일 인스타 콘텐츠를 자동으로 만들어준다"는 걸 이해시키기

**구성**:
- Hero 섹션:
  - 헤드라인: "매일 인스타 콘텐츠, 이제 AI가 알아서"
  - 서브헤드: "가게 정보만 알려주면, 매일 아침 8시 자동 생성"
  - CTA 버튼: "30일 무료로 시작하기" (Primary, 크고 눈에 띄게)
  - 실제 화면 프리뷰 (모바일 목업에 생성된 콘텐츠 예시)

- Features 섹션 (3개):
  1. AI 자동 생성 (이미지 + 캡션)
  2. 원클릭 인스타 발행
  3. 예약 발행 가능

- Pricing 섹션:
  - 무료 체험 30일 강조
  - 베이직 플랜: 29,000원/월
  - 프로 플랜: 49,000원/월
  - 가격 차이는 재생성 횟수, 예약 슬롯 수 등

- Footer: 간단한 링크, 문의 정보

**모바일 고려사항**:
- Hero 이미지는 세로로 길게 (스크롤 최소화)
- CTA 버튼 sticky 또는 반복 배치

---

### 2. 회원가입/로그인
**목표**: 가입 마찰 최소화

**구성**:
- Google 소셜 로그인 버튼 (Primary)
- 또는 이메일 로그인 (Secondary, 접히거나 작게)
- "30일 무료 체험, 카드 등록 없음" 문구 강조

**모바일 고려사항**:
- 버튼 높이 48px 이상
- 여백 충분히 (오타 방지)

---

### 3. 온보딩 1/2 - 가게 정보 입력
**목표**: 재미있게, 부담 없이 정보 수집

**구성**:
- 진행 바: "1/2" 또는 프로그레스 바
- 질문 형태로 친근하게:
  - "가게 이름이 뭐예요?"
    - 텍스트 입력 (큰 폰트)
  - "어떤 분위기의 가게인가요?" (최대 3개 선택)
    - 버튼 형태 키워드: [아늑한] [모던한] [빈티지] [감성적인] [미니멀] [화려한]
    - 선택 시 버튼 색상 변화 (Primary 색상)
  - "대표 메뉴 3개를 알려주세요"
    - 입력 필드 3개 (동적 추가 가능)
    - 예시: "아메리카노, 크로플, 에그타르트"

- 하단 고정 버튼: "다음" (Primary, 넓게)

**모바일 고려사항**:
- 키워드 버튼은 2줄로 배치 (손가락으로 누르기 편하게)
- 키보드 올라와도 다음 버튼 보이게

---

### 4. 온보딩 2/2 - 인스타 연동
**목표**: 인스타 연동의 가치 전달 + 불안감 해소

**구성**:
- 진행 바: "2/2"
- 설명 텍스트:
  - "인스타그램 계정을 연결하면"
  - "만든 콘텐츠를 바로 발행할 수 있어요"
  - "걱정 마세요, 우리는 절대 임의로 게시하지 않아요"

- Instagram 연동 버튼 (Instagram 브랜드 컬러)
  - 아이콘 + "Instagram 계정 연결하기"

- 하단 고정 버튼:
  - "나중에 하기" (Ghost button)
  - "연결하고 시작하기" (Primary, 연동 완료 후 활성화)

**모바일 고려사항**:
- OAuth 화면 전환 시 로딩 스피너
- 연동 실패 시 명확한 에러 메시지 + 재시도

---

### 5. 대시보드/홈 - 오늘의 콘텐츠
**목표**: "오늘 뭘 올릴지 이미 준비됐다"는 안도감

**레이아웃**:
- 상단: Navbar
  - 좌측: 로고 또는 가게 이름
  - 우측: 설정 아이콘, TrialBadge (남은 일수)

- 메인 콘텐츠 영역:
  - **ContentCard** (중앙 배치, 카드 형태)
    - 생성된 이미지 (정사각형, Instagram 비율)
    - 캡션 텍스트 (접히거나 펼침 가능)
    - 하단 액션 버튼:
      - "캡션 수정" (Secondary)
      - "이미지 재생성" (Secondary) + 재생성 횟수 표시 (3/5)

  - **발행 액션 영역** (카드 하단 또는 별도 섹션)
    - "즉시 발행" 버튼 (Primary, 크게)
    - "예약 발행" 버튼 (Secondary)

- 하단 또는 좌측: 네비게이션
  - 홈 (현재)
  - 히스토리
  - 설정

**모바일 고려사항**:
- 이미지는 화면 폭의 90% 정도 차지
- 하단 탭 네비게이션 (고정)
- 즉시 발행 버튼은 엄지 영역 (하단 중앙)

**상태별 UI**:
- 로딩 중: 스켈레톤 UI
- 생성 실패: 에러 메시지 + "다시 생성하기" 버튼
- 발행 완료: 축하 토스트 메시지 + 히스토리로 이동 제안

---

### 6. 캡션 수정 모달/화면
**목표**: 빠르게 수정하고 닫기

**구성**:
- 모달 또는 전체 화면 (모바일: 전체 화면 권장)
- 상단: "캡션 수정" 타이틀 + X 닫기 버튼
- 텍스트 영역 (Textarea, 자동 높이 조절)
- 글자 수 표시 (Instagram 제한 고려)
- 하단 고정 버튼:
  - "취소" (Ghost)
  - "완료" (Primary)

**모바일 고려사항**:
- 키보드 올라와도 버튼 보이게
- 자동 포커스

---

### 7. 이미지 재생성 플로우
**목표**: 횟수 제한 인지 + 빠른 재생성

**구성**:
- "이미지 재생성" 버튼 클릭 시:
  - 확인 모달: "이미지를 다시 만들까요? (남은 횟수: 3/5)"
  - "취소" / "재생성" 버튼
- 재생성 중: 로딩 스피너 + "새로운 이미지를 만들고 있어요..."
- 완료: 새 이미지로 교체 + 부드러운 애니메이션

**재생성 횟수 시각화**:
- 버튼 옆 또는 하단에 "3/5" 표시
- 0/5일 경우 버튼 비활성화 + "플랜 업그레이드" 링크

---

### 8. 예약 발행 화면
**목표**: 명확한 시간 선택 + 실수 방지

**구성**:
- SchedulePicker 컴포넌트:
  - 날짜 선택 (캘린더 또는 드롭다운)
  - 시간 선택 (시/분 선택기)
  - 선택된 시간 미리보기: "2026년 3월 20일 오후 2시 30분에 발행됩니다"

- 하단 고정 버튼:
  - "취소"
  - "예약 완료" (Primary)

**모바일 고려사항**:
- Native date/time picker 활용
- 선택 후 즉시 미리보기 업데이트

---

### 9. 발행 히스토리
**목표**: 지난 콘텐츠 관리 + 성과 확인

**구성**:
- 리스트 형태:
  - 썸네일 이미지 (작게)
  - 발행 날짜 + 시간
  - 상태 배지: "발행 완료" / "예약됨" / "실패"
  - 탭 메뉴: 클릭 시 상세 보기 (이미지 + 캡션 전체)

- 필터/정렬:
  - 전체 / 발행 완료 / 예약됨
  - 최신순 / 오래된순

**모바일 고려사항**:
- 무한 스크롤 또는 페이지네이션
- 썸네일은 리스트 좌측 작게 배치

---

### 10. 설정 화면
**목표**: 필요한 설정을 쉽게 찾고 수정

**구성**:
- 섹션별 카드 형태:
  1. **가게 정보**
     - 가게 이름, 분위기 키워드, 대표 메뉴
     - "수정" 버튼

  2. **인스타그램 연동**
     - 연동된 계정 표시: @계정명
     - "연동 해제" 버튼 (Destructive, 빨간색)
     - 연동 안 된 경우: "Instagram 계정 연결하기"

  3. **구독 관리**
     - 현재 플랜 표시 (무료 체험 / 베이직 / 프로)
     - 남은 체험 일수 (체험 중인 경우)
     - "플랜 변경" 버튼
     - 결제 정보 (카드 끝 4자리)

  4. **기타**
     - 로그아웃
     - 계정 삭제 (Destructive)

**모바일 고려사항**:
- 각 섹션 카드로 분리 (여백 충분)
- Destructive 액션은 확인 모달 필수

## 컴포넌트 시스템

### 컬러 팔레트 (Tailwind CSS 기준)

**Primary** (Brand Color - Instagram 연상 + 따뜻함):
- `primary-50`: #FFF5F5
- `primary-100`: #FFE3E3
- `primary-500`: #F87171 (메인)
- `primary-600`: #DC2626 (hover)
- `primary-700`: #B91C1C

**Secondary** (Neutral):
- `gray-50`: #F9FAFB
- `gray-100`: #F3F4F6
- `gray-500`: #6B7280
- `gray-700`: #374151
- `gray-900`: #111827

**Success** (발행 완료):
- `green-500`: #10B981

**Warning** (체험 종료 임박):
- `yellow-500`: #F59E0B

**Error** (연동 끊김, 에러):
- `red-500`: #EF4444

**Background**:
- 기본: `white` 또는 `gray-50`
- 카드: `white` + shadow

---

### 타이포그래피

**폰트**:
- 한글: Pretendard Variable (시스템 폰트 fallback: -apple-system, BlinkMacSystemFont, "Segoe UI")
- 영문/숫자: Inter

**크기**:
- Hero 헤드라인: `text-4xl` (36px) ~ `text-5xl` (48px)
- 페이지 타이틀: `text-2xl` (24px) ~ `text-3xl` (30px)
- 본문: `text-base` (16px)
- 캡션/부가 정보: `text-sm` (14px)
- 버튼 텍스트: `text-base` (16px) font-semibold

**행간**:
- 타이틀: `leading-tight` (1.25)
- 본문: `leading-relaxed` (1.625)

---

### 핵심 컴포넌트

#### 1. Button
- **Primary**: `bg-primary-500 hover:bg-primary-600 text-white`
- **Secondary**: `bg-white border-2 border-gray-300 hover:border-gray-400 text-gray-700`
- **Ghost**: `bg-transparent text-gray-700 hover:bg-gray-100`
- **Destructive**: `bg-red-500 hover:bg-red-600 text-white`
- 높이: `h-12` (48px) 모바일 기본
- 둥근 모서리: `rounded-lg` (8px)

#### 2. Card (ContentCard, 각종 카드)
- 배경: `bg-white`
- 그림자: `shadow-md hover:shadow-lg` (인터랙티브한 경우)
- 모서리: `rounded-xl` (12px)
- 패딩: `p-4` ~ `p-6`

#### 3. Input / Textarea
- 높이: `h-12` (48px)
- 테두리: `border-2 border-gray-300 focus:border-primary-500`
- 모서리: `rounded-lg`
- 패딩: `px-4`

#### 4. Badge (TrialBadge, 상태 배지)
- 크기: `text-xs` (12px) ~ `text-sm` (14px)
- 패딩: `px-2 py-1`
- 모서리: `rounded-full`
- 색상:
  - 체험 중: `bg-yellow-100 text-yellow-800`
  - 발행 완료: `bg-green-100 text-green-800`
  - 예약됨: `bg-blue-100 text-blue-800`
  - 실패: `bg-red-100 text-red-800`

#### 5. Modal
- 배경 오버레이: `bg-black/50` (50% 투명도)
- 모달 본체: `bg-white rounded-xl shadow-2xl`
- 최대 너비: `max-w-md` (모바일: 풀 스크린)
- 애니메이션: fade-in + slide-up

#### 6. Toast (알림 메시지)
- 위치: 상단 중앙 또는 하단 (모바일: 하단 권장)
- 배경: `bg-gray-900 text-white` (성공: `bg-green-500`)
- 모서리: `rounded-lg`
- 자동 사라짐: 3초

#### 7. Loading Spinner
- 크기: `w-8 h-8` (32px)
- 색상: `border-primary-500`
- 애니메이션: `animate-spin`

#### 8. Skeleton UI
- 배경: `bg-gray-200 animate-pulse`
- 모서리: 원본 컴포넌트와 동일
- 사용처: 이미지 로딩, 텍스트 로딩

---

### 간격 시스템 (Spacing)
- 섹션 간: `space-y-8` (32px) ~ `space-y-12` (48px)
- 카드 간: `space-y-4` (16px)
- 요소 간: `space-y-2` (8px)
- 여백 (패딩): `p-4` (16px) ~ `p-6` (24px)
- 컨테이너 좌우 여백 (모바일): `px-4` (16px)

---

### 그림자 (Shadow)
- 카드: `shadow-md` (기본)
- 호버: `hover:shadow-lg`
- 모달: `shadow-2xl`
- 네비게이션 바: `shadow-sm`

### 애니메이션
- 버튼 호버: `transition-colors duration-200`
- 카드 호버: `transition-shadow duration-200`
- 모달 등장: `transition-all duration-300`
- 페이지 전환: Next.js 기본 전환 또는 Framer Motion (선택)

## 모바일 우선 고려사항

### 1. 터치 영역
- 최소 높이: 48px (Apple HIG 권장)
- 버튼 간 간격: 최소 8px
- 중요 버튼 (CTA): 56px~64px 높이 권장

### 2. 엄지 영역 (Thumb Zone)
- 핵심 액션 버튼 (즉시 발행, 예약 발행): 화면 하단 중앙
- 네비게이션: 하단 고정 탭 바
- 위험한 액션 (삭제, 연동 해제): 상단 또는 모달 내부

### 3. 세로 스크롤 최소화
- 한 화면에 핵심 정보 모두 표시
- 긴 콘텐츠는 접기/펼치기
- 대시보드/홈: 스크롤 없이 이미지 + 버튼 모두 보이게

### 4. 가로 모드 (선택 사항)
- MVP에서는 세로 모드 최적화에 집중
- 가로 모드는 자동 적응으로 충분

### 5. 키보드 대응
- Input 포커스 시 버튼이 키보드에 가려지지 않게
- 필요 시 `scrollIntoView()` 사용

### 6. 로딩 시간
- 이미지 로딩: 스켈레톤 UI + 저해상도 → 고해상도 점진적 로딩
- API 호출: 낙관적 UI 업데이트 (즉시 반영 → 실패 시 롤백)

### 7. 오프라인 고려 (선택 사항)
- MVP: 온라인 전용
- 추후: Service Worker로 오프라인 히스토리 조회
