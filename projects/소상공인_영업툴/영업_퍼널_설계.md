# 영업 퍼널 설계 — 오픈 to 클로징
> 진단 데이터를 단계별로 압축해서 전달하는 구조

---

## 전체 흐름

```
[1단계] 콜드 DM        → 1개 숫자로 충격   → 목표: 답장
[2단계] 첫 전화        → 3개 문제로 공감   → 목표: 미팅 약속
[3단계] 심층 상담      → 전체 진단 공개    → 목표: "이거 고쳐야겠다"
[4단계] 제안 미팅      → PPT + 패키지      → 목표: 계약
[5단계] 클로징/팔로업  → 이의 처리         → 목표: 서명
```

---

## 데이터 압축 원칙

단계가 올라갈수록 데이터를 **풀어준다**.

| 단계 | 보여주는 데이터 | 이유 |
|------|----------------|------|
| 콜드 DM | **1개** (가장 충격적인 숫자) | 너무 많으면 광고로 인식 |
| 첫 전화 | **3개** (점수 + TOP2 문제) | 관심은 생겼는데 아직 신뢰 없음 |
| 심층 상담 | **7~8개** (체크리스트 + 경쟁사) | 신뢰 형성됨, 깊이 있는 대화 가능 |
| 제안 미팅 | **전체** (PPT 11장) | 계약 직전, 모든 근거 제시 |

## 1단계 — 콜드 DM

### 목표
답장 1개 받기. 계약 얘기 꺼내지 마라.

### 어떤 숫자 1개를 꺼낼지

업체 진단 결과에서 아래 우선순위로 가장 충격적인 것 1개 선택:

```
우선순위 1: 경쟁사 리뷰 격차
  → 조건: 내 리뷰가 경쟁사 평균의 30% 이하일 때
  → 숫자: "이 지역 상위 업체 평균 {경쟁사평균}개 vs 사장님 {내리뷰}개"

우선순위 2: 순위 + 잃고 있는 고객 수
  → 조건: 순위 10위 밖이고 월 검색량 500 이상일 때
  → 숫자: "지금 {순위}위라서 한 달에 {손실고객}명 정도 경쟁사로 가고 있어요"

우선순위 3: 방치된 핵심 항목
  → 조건: 새소식 90일 이상 없거나, 저장수 0이거나, 사장님 답글 0%
  → 숫자: "새소식이 {N}일째 없어서요" / "저장 수가 0이에요"

우선순위 4: 종합 등급
  → 조건: D등급 (39점 이하)
  → 숫자: "네이버 플레이스 진단해봤더니 100점 만점에 {점수}점 나왔어요"
```

### DM 템플릿 (업체 상황별)

**[A형] 리뷰 격차가 심할 때**
```
[업체명] 사장님, 안녕하세요.
저희가 이 지역 미용실들 플레이스 현황 분석하다가 연락드렸는데요.
이 동네 상위 업체들 리뷰 평균이 {경쟁사평균}개인데
사장님 가게는 {내리뷰}개 있으시더라고요.
무료로 전체 진단 한번 드려도 될까요? 5분이면 돼요.
```

**[B형] 순위/노출 문제일 때**
```
[업체명] 사장님, 안녕하세요.
'{메인키워드}' 검색하면 지금 {순위}위로 뜨시는데,
이 위치에서는 한 달에 클릭을 {현재클릭추정}번 정도밖에 못 받으세요.
1위 업체는 같은 키워드로 {1위클릭추정}번 받고 있고요.
무료 진단 한번 드려도 될까요?
```

**[C형] 방치 항목이 있을 때**
```
[업체명] 사장님, 안녕하세요.
플레이스 보다가 새소식이 {N}일째 없으신 걸 봤는데요.
네이버가 요즘 이걸 되게 중요하게 보거든요.
혹시 이 부분 포함해서 무료 진단 한번 드려도 될까요?
```

### 발송 규칙
- 업체명 + 숫자 1개 반드시 포함 (개인 연락처럼 느껴지게)
- 150자 이내
- 마지막은 반드시 질문형 ("~드려도 될까요?")
- 가격/서비스 이름 절대 언급 금지

---

## 2단계 — 첫 전화 (답장 왔을 때)

### 목표
미팅 약속 잡기. 진단 결과를 "맛보기"만 보여준다.

### 보여주는 데이터 3개

```
① 종합 점수 + 등급
② 가장 심각한 취약 항목 1~2개 (숫자 포함)
③ 경쟁사 대비 격차 (비율로)
```

### 전화 스크립트

```
"안녕하세요 사장님, 아까 문자 드렸던 [이름]이에요.

저희가 사장님 플레이스 간단히 분석해봤는데,
전체 점수가 95점 만점에 {점수}점 나왔거든요. {등급}등급이에요.

제일 아쉬운 게 두 가지인데요,
첫 번째로 {취약항목1} 부분이 {현재값}밖에 없으세요.
이 지역 상위 업체들은 평균 {경쟁사값}이거든요.

두 번째가 {취약항목2}인데, 이게 요즘 네이버 알고리즘에서
되게 중요하게 보는 항목인데 지금 아예 없으셔서요.

나머지 진단 결과도 있는데, 한 번 직접 보여드리면 어떨까요?
30분이면 충분해요. 이번 주 언제 괜찮으세요?"
```

### 이 단계에서 답하면 안 되는 질문
- "얼마예요?" → "진단 먼저 보여드리고 나서 말씀드릴게요. 업체마다 달라서요."
- "뭐 해주는 거예요?" → "직접 보여드리는 게 빠를 것 같아서요. 30분만 주시면요."
- "그거 해서 효과 있어요?" → "저희가 진단한 업체 중에 {사례} 있었는데, 그것도 같이 보여드릴게요."

---

## 3단계 — 심층 상담 (미팅 전 사전 준비용)

### 목표
미팅 전에 상대가 이미 "문제가 있다"는 걸 인식하게 만들기.
미팅 당일 "왜 필요한지"가 아니라 "어떻게 고칠지"를 얘기할 수 있게.

### 사전에 보내도 되는 데이터 (카톡/문자 2차 발송)

미팅 하루 전날, 아래 형식으로 데이터 요약 텍스트 발송:

```
[업체명] 사장님, 내일 미팅 전에 간단히 공유드릴게요.

📊 진단 요약
• 종합 점수: {점수}/95점 ({등급}등급)
• 이 지역 상위 5개 업체 평균 대비 {상대점수}% 수준

❌ 현재 비어있는 항목
• {취약항목1}: {현재값} (기준 {기준값})
• {취약항목2}: {현재값} (기준 {기준값})
• {취약항목3}: {현재값} (기준 {기준값})

내일 자세히 설명드릴게요. 내일 {시간}에 뵐게요!
```

### 보여주는 데이터 목록 (7~8개)
1. 종합 점수 / 등급
2. 경쟁사 평균 대비 %
3. 취약 항목 TOP 3 (숫자 포함)
4. 방문자 리뷰 수 vs 경쟁사 평균
5. 사진 수 vs 경쟁사 평균
6. 저장 수
7. 새소식 마지막 날짜
8. 사장님 답글율

---

## 4단계 — 제안 미팅 (PPT 발표)

### 목표
"이거 해야겠다" 느끼게 하기. 계약서 들이밀 준비.

### 슬라이드 발표 순서 + 멘트

**슬라이드 1 (표지)**
> "오늘 [업체명] 사장님을 위해 준비한 자료예요."

**슬라이드 2 (종합 점수)**
> "{점수}점, {등급}등급이에요. 근데 이게 단순한 점수가 아니라,
> 지금 경쟁사 대비 얼마나 뒤처져 있는지를 숫자로 만든 거거든요.
> *(잃고 있는 고객 수 있으면)* 지금 이 순위에서는 매달 {손실고객}명 정도가
> 사장님 가게 대신 다른 데 가고 있는 거예요."

**슬라이드 3 (경쟁사 비교 차트)**
> "이 차트가 제일 중요해요. 왼쪽이 사장님, 오른쪽이 이 지역 상위 업체 평균이에요.
> 리뷰 {내리뷰}개 vs {경쟁사평균}개. 이게 그냥 숫자 차이가 아니라
> 네이버가 이걸 보고 누구를 위에 올려줄지 결정하거든요."

**슬라이드 4 (업체 기본정보 + 순위)**
> "키워드 순위 확인하셨어요? '{메인키워드}'로 지금 {순위}위예요.
> 1위 업체는 같은 키워드로 한 달에 {1위클릭}번 클릭 받는데
> 사장님은 {내클릭}번 받으시고 있어요."

**슬라이드 5 (확장 키워드)**
> "그리고 이 키워드들도 있어요. 사장님 업체랑 연관된 키워드인데,
> 여기서 잡히는 게 없으면 이 고객들은 다 경쟁사로 가는 거예요."

**슬라이드 6 (스크린샷)**
> "이게 실제로 검색했을 때 화면이에요. 사장님 가게가 어디 있는지 보이시죠?"

**슬라이드 7 (22항목 체크리스트)**
> "빨간 X 보이시는 거 이게 지금 비어있는 항목들이에요.
> 이것들이 하나하나 네이버 알고리즘에서 점수로 들어가요.
> 특히 {취약항목1}이랑 {취약항목2} 이거 두 개가 제일 급해요."

**슬라이드 8 (취약 항목 TOP 3)**
> "이 세 가지만 집중적으로 고쳐도 순위가 다르게 나와요.
> 저희가 실제로 비슷한 상황에서 [사례 언급]한 적 있거든요."

**슬라이드 9 (패키지 소개)**
> "고칠 수 있는 방법은 세 가지 플랜이 있어요..."

**슬라이드 10 (도입 근거)**
> "왜 직접 하시기 어렵냐면요..."

**슬라이드 11 (견적서)**
> "사장님 상황에서는 {추천패키지}가 맞을 것 같아요.
> 근데 가장 중요한 건, 이거 안 하시면 매달 {손실고객}명이 계속 경쟁사로 가는 거예요.
> 그 분들 한 번 방문 가치가 평균 {객단가}원이라고 보면..."

### 클로징 멘트
```
"오늘 자료 보시면서 어떠셨어요?
저희가 이번 달 시작하시면 다음 달 리포트까지 드리는데,
계약서 오늘 쓰시면 첫 달 세팅비 없이 진행해드릴 수 있어요.
어떻게 생각하세요?"
```

## 5단계 — 이의 처리 + 클로징

### 자주 나오는 반응 5가지

**"생각해볼게요"**
```
"네, 물론이죠. 근데 사장님, 생각하시는 동안에도 매달 {손실고객}명은 경쟁사로 가고 있거든요.
다음 주까지 결정하시면 이번 달 세팅 잡아드릴 수 있어요. 언제 다시 연락드릴까요?"
```

**"비싼 것 같아요"**
```
"그 마음 이해해요. 근데 {추천패키지} 월 {금액}원인데,
이게 고객 {N}명만 더 와도 본전이에요.
사장님 업종 평균 객단가가 {객단가}원이면, 한 달에 {몇명}명만 더 오시면 되거든요."
```

**"효과가 있을까요"**
```
"저도 100% 보장은 못 드리는데요, 저희가 진단한 업체들 중에
비슷한 상황에서 3개월 만에 순위가 {N위} 올라간 곳이 있어요.
일단 3개월만 해보시고, 효과 없으면 그때 말씀해주세요."
```

**"직접 해볼게요"**
```
"물론 하실 수 있죠. 근데 아까 체크리스트 보셨죠?
저장 수, 새소식, 영수증리뷰, 해시태그... 이거 하나하나 다 시간이 걸리거든요.
사장님이 직접 하시는 시간 값을 따지면 오히려 저희가 더 싼 거예요."
```

**"나중에 해요"**
```
"나중이라는 게 언제쯤이에요? 경쟁사들이 지금도 올리고 있어서요.
3개월 뒤에 격차가 더 벌어지면 그때는 따라잡는 데 더 오래 걸리거든요."
```

## 단계별 데이터 요약표

| 단계 | 전달 방식 | 꺼내는 데이터 | 꺼내지 않는 데이터 |
|------|-----------|--------------|-------------------|
| 콜드 DM | 문자/카톡 | 충격 숫자 1개 | 점수, 패키지, 가격 전부 |
| 첫 전화 | 전화 | 점수+등급, 취약 TOP2, 경쟁사 격차 | 전체 체크리스트, 가격 |
| 사전 문자 | 카톡 | 점수, 취약 TOP3, 경쟁사 % | 가격, 패키지 |
| 제안 미팅 | PPT | 전체 데이터 + 기회비용 + 패키지 | 없음 (다 꺼냄) |
| 클로징 | 대화 | 기회비용 + 손실 계산 + 혜택 | 없음 |

---

## 업체 진단 결과 → 콜드 DM 자동 선택 로직

```
IF 경쟁사 리뷰 평균 / 내 리뷰 >= 5배:
    → A형 DM (리뷰 격차)
ELIF 순위 > 10 AND 월 모바일 검색량 >= 500:
    → B형 DM (순위 + 손실 고객)
ELIF 새소식 >= 90일 OR 저장수 == 0 OR 사장님답글율 == 0:
    → C형 DM (방치 항목)
ELSE:
    → A형 DM (기본형, 리뷰 기준)
```

---

## 이 퍼널에서 가장 중요한 원칙

1. **각 단계에서 딱 하나만 얻으려 해라**
   - DM: 답장 1개 / 전화: 미팅 약속 1개 / 미팅: 계약 1개

2. **데이터는 사실처럼, 공감은 감정으로**
   - 숫자 → 사실로 전달 ("리뷰가 {N}개예요")
   - 의미 → 감정으로 전달 ("이 분들이 매달 경쟁사로 가고 있는 거예요")

3. **절대 먼저 가격 얘기 꺼내지 마라**
   - 가격은 반드시 상대가 먼저 묻거나, 미팅 슬라이드 9번부터

4. **손실 숫자가 가격보다 커야 계약된다**
   - "월 {손실고객}명 × 객단가 = {손실금액}원" > "월 {패키지금액}원"
   - 이 계산이 머릿속에 자리잡혀야 계약 가능
