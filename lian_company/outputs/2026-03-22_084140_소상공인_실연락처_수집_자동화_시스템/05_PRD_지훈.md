# PRD: 소상공인 실연락처 수집 자동화 시스템

**작성자**: 지훈 (실행팀 PRD 담당)
**버전**: v1.0
**작성일**: 2025-01-27
**단계**: 내부용 MVP (Phase 1)

---

## 제품 개요

마케팅 에이전시가 구글 비즈니스 프로필·네이버 블로그·당근마켓·인스타그램에서 소상공인의 실연락처를 자동 수집하고 안심번호를 제거한 뒤 업종별 CSV로 출력하는 수집 전용 내부 툴.

---

## Must Have

1. **멀티소스 수집**: 구글 비즈니스 프로필, 네이버 블로그, 당근마켓 업체 프로필, 인스타그램 비즈니스 계정 — 4개 플랫폼에서 전화번호·카카오톡 채널·이메일 자동 수집
2. **안심번호(050) 자동 필터링**: 수집된 전화번호 중 050 시작 번호를 자동 감지하여 제외 처리, 필터링된 건수 별도 카운트 표시
3. **업종별 자동 분류**: 수집 시 지정한 키워드(업종명, 지역명) 기반으로 결과를 카테고리 컬럼에 자동 태깅 (예: 음식점 / 뷰티 / 인테리어 등)
4. **수집 작업 설정 UI**: 플랫폼 선택, 검색 키워드, 지역, 수집 건수 상한 설정 후 작업 실행 — 웹 기반 대시보드
5. **중복 제거**: 동일 전화번호·이메일이 여러 플랫폼에서 수집될 경우 중복 제거 후 출처 플랫폼 병기
6. **CSV 출력**: 업체명 / 업종 / 전화번호 / 카카오톡 / 이메일 / 수집 출처 / 수집일시 컬럼 포함 CSV 다운로드
7. **수집 이력 관리**: 작업 단위로 수집 이력(작업명, 실행일시, 수집 건수, 필터링 건수, 상태) 목록 저장 및 재다운로드

---

## Must NOT (범위 외)

- **자동 발송 기능 없음**: 이메일·문자·카카오 메시지 발송 기능 일절 포함 금지
- **실시간 모니터링 없음**: 신규 업체 등록 감지, 알림 푸시 등 모니터링 자동화 제외
- **CRM 연동 없음**: Salesforce, HubSpot 등 외부 CRM API 연동 Phase 1 제외
- **유료 결제 시스템 없음**: Phase 1은 내부용이므로 구독·결제 플로우 제외
- **멀티테넌트 없음**: 외부 에이전시 계정 분리 구조 Phase 1 제외 (단일 조직 사용)
- **분석/리포트 대시보드 없음**: 수집 데이터 기반 통계 시각화 제외
- **공공데이터 API 연동 없음**: 사업자등록 정보 등 외부 API 연동 Phase 1 제외

## User Flow

```
[사전 조건] 로그인된 내부 사용자
```

**1단계**: 대시보드 접속 → 수집 작업 목록 화면 확인 (이전 작업 이력 표시)

**2단계**: \[새 작업 만들기] 버튼 클릭 → 작업 설정 화면 이동

**3단계**: 작업 설정 입력
- 작업명 입력 (예: "강남구 카페 2025-01")
- 수집 플랫폼 선택 (1개 이상 체크박스)
- 검색 키워드 입력 (예: "강남구 카페")
- 지역 입력 (선택, 예: "서울 강남구")
- 수집 건수 상한 입력 (기본값: 100건)

**4단계**: \[수집 시작] 버튼 클릭 → 작업 대기열 등록 확인 토스트 메시지 표시

**5단계**: 작업 목록 화면에서 진행 상태 확인 (대기 중 / 수집 중 / 완료 / 오류)
- 수집 중: 진행 건수 실시간 표시 (예: "47/100건")

**6단계**: 상태 "완료" 확인 → 해당 작업 행 클릭 → 수집 결과 상세 화면 이동

**7단계**: 결과 상세 화면에서 확인
- 총 수집 건수 / 050 필터링 제거 건수 / 최종 유효 건수 요약 카드 확인
- 결과 테이블 미리보기 (최대 50행 표시)
- 업종별 건수 요약 표 확인

**8단계**: \[CSV 다운로드] 버튼 클릭 → 로컬에 파일 저장

**9단계**: 다운로드한 CSV를 열어 영업팀에 전달 → 영업팀이 직접 연락

| 화면명 | 설명 | 핵심 요소 |
|--------|------|-----------|
| **로그인** | 내부 사용자 인증 | 이메일+비밀번호 입력, 로그인 버튼, 에러 메시지 |
| **작업 목록 (대시보드)** | 전체 수집 작업 이력 조회 | 작업명 / 생성일시 / 상태 배지 / 유효 건수 / 액션(상세보기·삭제) 테이블, [새 작업 만들기] 버튼, 상태별 필터 탭 |
| **작업 설정** | 새 수집 작업 파라미터 입력 | 작업명 텍스트 인풋, 플랫폼 멀티체크박스(구글/네이버/당근/인스타), 키워드 인풋, 지역 인풋, 수집 건수 상한 넘버 인풋, [수집 시작] 버튼, [취소] 버튼 |
| **수집 결과 상세** | 완료된 작업의 수집 결과 확인 및 다운로드 | 요약 카드 3개(총 수집/050 제거/유효 건수), 업종별 건수 요약 테이블, 결과 미리보기 테이블(업체명·업종·전화번호·카카오·이메일·출처·수집일시), 페이지네이션, [CSV 다운로드] 버튼 |
| **오류 상세** | 수집 실패 작업의 오류 내역 확인 | 오류 발생 플랫폼·시각·메시지 목록, [재시도] 버튼, [대시보드로 돌아가기] 버튼 |

## 성공 기준

**수집 품질**
- 플랫폼별 수집 성공률 ≥ 80% (요청 건수 대비 실제 수집 건수)
- 050 안심번호 필터링 정확도 ≥ 95% (수동 샘플 검수 100건 기준)
- 중복 제거 정확도 ≥ 99% (동일 번호 이중 출력 건수 0건 목표)

**운영 효율**
- 작업 1회 실행 시 100건 수집 소요 시간 ≤ 10분
- 수집 결과 CSV 생성 후 다운로드까지 ≤ 3초

**내부 사용 지표 (3개월 내)**
- 주 3회 이상 작업 실행 유지
- 수집 데이터 기반 영업팀 연락 시도율
