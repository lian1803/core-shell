## 관련 트렌드

1. **Web Scraping + AI 크롤링의 규제 강화** ⭐
   - Google, Meta 등 TOC 변경으로 자동화 수집 제약 증가
   - 법적 리스크(개인정보보호법, 정보통신망법) 심화

2. **B2B 데이터 수집 SaaS의 폭발적 성장** ⭐
   - Apollo.io, Hunter.io 같은 해외 서비스 급성장
   - 국내: 데이터 수집/정제 자동화 툴 수요 폭증

3. **직접 발송 기반 마케팅 오토메이션**
   - 스팸 필터 우회 목적의 "사람이 보내는" 형태 선호
   - 개인 계정/매니저 기반 발송 트렌드

4. **Platform API vs Scraping 이분화**
   - 공식 API 의존도 증가 (Naver Business, Google My Business API)
   - 비공식 크롤링은 점점 위험해지는 추세

5. **마케팅 에이전시의 인하우스 툴화**
   - 프롭테크, 에드테크처럼 에이전시 자체 솔루션화 트렌드

---

## 리안에게 의미 있는 것

**수집 자동화는 핫하지만, 법적/기술적 장벽이 급상승 중.** 지금 내부용으로 검증하면서 규제 변화를 미리 감지하고, SaaS 전환 전에 API 기반으로 피봇할 준비가 필요. 단순 스크래핑은 6개월 뒤 막힐 수 있음.

## 활용 제안

### 1️⃣ 즉시 (내부용)
- **스택 로드맵**: Selenium → Playwright → 공식 API 순으로 우선순위 정렬
- 네이버/구글 비즈니스 프로필은 공식 API 우선 검토 (법적 보호)
- 당근마켓/인스타는 API 불가 구간이므로, 수집 시 이용약관 명시적 검토

### 2️⃣ SaaS 판매 전 (리스크 회피)
- **컴플라이언스 먼저**: 변호사 검수 (마크업/수집 경계선)
- 고객에게 "발송 책임은 고객" 명시 계약서
- 수집 데이터 업데이트 주기 명시 (신선도 > 신뢰도)

### 3️⃣ 차별화 포인트
- 안심번호 필터링 + 업종 자동 분류는 경쟁사 대비 강점
- → "검증된 실연락처" 마케팅 메시지로 공략 (정확도 수치화)
- 에이전시 → SMB 마케터 확장 시, "법적으로 안전한 수집" 강조

### 4️⃣ 모니터링
- HN/Reddit의 데이터 프라이버시 관련 뉴스 실시간 추적
- 각 플랫폼 ToC 변경사항 월 1회 체크 (Scraper
