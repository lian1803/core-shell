# PRD: 네이버 플레이스 자동 진단 + PPT 제안서 생성 툴

**작성자:** 지훈 (실행팀 PRD)
**버전:** v1.0
**상태:** 개발 착수 가능

---

## 제품 개요
업체명 하나 입력하면 네이버 플레이스 데이터를 자동 크롤링해서 영업용 5페이지 PPT 제안서를 즉시 뽑아주는 마케팅 대행사 개인 툴

---

## Must Have

1. **업체명 검색 + 플레이스 매칭** — 업체명 입력 시 네이버 플레이스 검색 결과에서 해당 업체를 자동 식별 및 URL 매칭
2. **플레이스 데이터 자동 수집** — 사진 수, 리뷰 수(영수증/방문), 블로그 리뷰 수, 대표 키워드 노출 순위, 영업시간·메뉴 정보 완성도 수집
3. **진단 점수 자동 산출** — 수집 데이터를 기반으로 항목별(사진/리뷰/블로그/키워드) 점수화 및 종합 등급 산출 (A~D 또는 0~100점)
4. **키워드 검색량 조회** — 네이버 검색광고 API 연동하여 대표 키워드 3~5개 월간 검색량 자동 조회 (크롤링 Plan B)
5. **5페이지 PPT 자동 생성** — python-pptx 기반, 고정 템플릿에 데이터를 자동 삽입하여 .pptx 파일 다운로드
6. **생성 이력 저장** — 업체명 + 생성일시 + 진단 결과 로컬 DB 저장 (재생성 및 히스토리 조회 가능)

---

## Must NOT (범위 외)

- 로그인 / 회원가입 / 다중 사용자 지원 → 개인 툴, 단일 사용자 전제
- 실시간 모니터링 / 주기적 자동 알림 → MVP 이후 검토
- 경쟁사 비교 분석 → MVP 이후 검토
- PDF 출력 / 웹 공유 링크 생성 → pptx 다운로드만
- 맞춤 컨설팅 / 프리미엄 보고서 기능 → 전략팀 지시에 따라 제거
- PPT 템플릿 커스터마이징 UI → 템플릿 고정, 코드 레벨 수정만
- 모바일 지원 → 데스크탑 웹 전용

## User Flow

```
1단계: 툴 접속 (localhost 또는 단일 URL)
        ↓
2단계: 업체명 입력 + [진단 시작] 버튼 클릭
        ↓
3단계: 네이버 플레이스 자동 검색 → 매칭 업체 1~3개 결과 노출
        ↓
4단계: 사용자가 정확한 업체 선택 (업체명 + 주소로 구분)
        ↓
5단계: 자동 데이터 수집 진행 (로딩 상태 표시, 예상 5~15초)
        → 수집 항목: 사진 수 / 리뷰 수 / 블로그 수 / 키워드 순위 / 정보 완성도 / 키워드 검색량
        ↓
6단계: 진단 결과 화면 출력
        → 항목별 점수 + 종합 등급 + 개선 포인트 텍스트 자동 생성
        ↓
7단계: [PPT 생성] 버튼 클릭
        ↓
8단계: .pptx 파일 자동 다운로드
        ↓
9단계: 생성 이력 자동 저장 → 이력 목록에서 재다운로드 가능
```

**예외 처리 분기:**
- 3단계에서 업체 미발견 → "업체를 찾을 수 없습니다. 정확한 상호명을 입력해주세요" 안내
- 5단계에서 크롤링 차단 감지 → "일부 항목을 불러오지 못했습니다. 직접 입력 모드로 전환하시겠습니까?" 팝업 → 수동 입력 후 PPT 생성 가능

## 화면 목록

| 화면명 | 설명 | 핵심 요소 |
|--------|------|-----------|
| **메인 / 검색 화면** | 툴 진입 시 첫 화면. 업체명 입력창이 중앙에 위치 | 업체명 텍스트 입력창, [진단 시작] 버튼, 최근 생성 이력 3건 미리보기 |
| **업체 선택 화면** | 검색 결과 매칭 업체 목록 표시 | 업체명 + 주소 + 카테고리 리스트 (라디오 버튼 선택), [이 업체로 진행] 버튼 |
| **데이터 수집 중 화면** | 크롤링 진행 상태 표시 | 항목별 수집 진행 표시 (체크리스트 형태: ✅ 사진 수집 완료 / ⏳ 리뷰 수집 중…), 예상 잔여 시간 |
| **진단 결과 화면** | 수집 완료 후 분석 결과 출력 | 항목별 점수 카드 (사진/리뷰/블로그/키워드/정보완성도), 종합 등급 배지, 개선 포인트 3줄 요약 텍스트, [PPT 생성] 버튼, [다시 검색] 버튼 |
| **수동 입력 화면** | 크롤링 실패 시 폴백. 사용자가 직접 데이터 입력 | 항목별 숫자 입력 필드 (사진 수, 리뷰 수, 블로그 수 등), [분석 실행] 버튼 |
| **이력 목록 화면** | 과거 생성 제안서 목록 조회 | 업체명 / 생성일시 / 종합 등급 테이블, [재다운로드] 버튼, [삭제] 버튼 |

## PPT 5페이지 구성 (템플릿 스펙)

개발자가 python-pptx로 구현할 정확한 슬라이드 구성:

| 페이지 | 제목 | 내용 |
|--------|------|------|
| **1페이지** | 표지 | 업체명, "네이버 플레이스 진단 제안서", 생성일, 대행사명(고정값) |
| **2페이지** | 현황 진단 요약 | 종합 등급, 항목별 점수 게이지 바 4개 (사진/리뷰/블로그/키워드), 한 줄 총평 |
| **3페이지** | 항목별 상세 분석 | 각 항목 현재 수치 + 업계 평균 비교 + 문제점 텍스트 |
| **4페이지** | 개선 제안 | 우선순위 TOP 3 액션 아이템 + 예상 효과 텍스트 |
| **5페이지** | 키워드 기회 분석 | 대표 키워드 3~5개 + 월간 검색량 + 현재 순위 + 노출 목표 순위 |

## 성공 기준

- **기능 완성도:** 업체명 입력 → PPT 다운로드까지 전체 플로우 오류 없이 동작 (성공률 90% 이상 / 테스트 20건 기준)
- **처리 속도:** 업체 선택 후 PPT 생성 완료까지 **30초 이내
