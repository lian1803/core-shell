## 시장 규모
한국 소상공인 SNS 자동화 시장은 소상공인 약 700만 개 업체 중 SNS 의존 업종이 많아 성장 잠재력 높음. AI-Social Studio 등 플랫폼 개발 착수로 시장 확대 중이며, 소셜비즈 고객 2배 증가(2024년 기준)로 **연평균 20-30% 성장 추정**.

## 경쟁사 현황
| 서비스 | 특징 | 약점 |
|--------|------|------|
| AI-Social Studio | 업종·제품 분석 후 블로그/인스타 자동 포스팅, 이미지·해시태그 생성, 예약 발행, 효과 분석 | 가격 미공개(착수 단계), 승인 필요로 완전 자동화 미흡, DM 자동화 미언급 |

## 타겟 페르소나
**30-50대 소상공인 사장님(카페·네일샵·식당 운영자)**: 매일 12시간 이상 가게 운영, 직원 1-3명, 연 매출 5천만~2억원. SNS 홍보 필요성 인지하나 시간 부족으로 포기. 월 마케팅 예산 5-10만원 내외.

## 핵심 Pain Point
1. 매일 인스타 콘텐츠(글·이미지) 제작 루틴이 시간 소모적(사진찍기→글쓰기→디자인→발행)
2. 꾸준한 게시 미루다 계정 정체, 팔로워·매출 증가 실패
3. 해시태그·전략 모름, 효과 분석 어려움
4. DM·댓글 수동 응대 부담, 판매 기회 놓침
5. 여러 툴(챗GPT·Canva·인스타) 연동 귀찮음

## 기회 포인트
**완전 자동화(업종 선택→매일 캡션+이미지 생성+인스타 직발행)**: 경쟁사 대부분 승인·수동 개입 필요하나, 월 9,900원 저가로 "셋팅만 하고 끝" 차별화. DM 자동화 결합 시 소셜비즈 보완, 릴스 자동 생성 추가로 업종 특화(카페 메뉴 이미지 AI 생성).

### [하은] 검증/반론 — Gemini 2.5 Flash

## 주요 반론

1. **시장 규모 및 성장률 추정의 불명확성**
   서윤의 "연평균 20-30% 성장 추정"은 유사 서비스의 성장률을 바탕으로 한 추정이며, '완전 자동화 콘텐츠 생성 및 발행'이라는 특정 니치 시장의 직접적인 수요나 규모를 나타내지 않음. 700만 소상공인 중 잠재 고객 비율, 실제 유료 전환율, 이탈률 등에 대한 근거 부족.

2. **핵심 기능 '인스타그램 자동 발행'의 기술적 난이도 및 정책 리스크**
   Meta는 무분별한 자동 포스팅 및 스팸 방지를 위해 서드파티 앱의 게시물 자동 발행에 대한 통제를 강화하고 있으며, 특히 이미지와 릴스의 완전 자동 발행은 제한적이거나 특정 승인된 파트너에게만 허용될 가능성이 높음. 경쟁사들(AI-Social Studio)도 "승인 필요로 완전 자동화 미흡"이라는 약점을 가지고 있음.

3. **AI 콘텐츠 생성 품질 및 유지 비용의 현실성**
   월 9,900원으로 '매일' '업종 맞춤' '고품질' 캡션과 이미지를 AI로 자동 생성하여 제공하는 것은 콘텐츠 품질 유지 비용(GPT 및 AI 이미지 생성 API 사용료) 측면에서 수익성이 낮을 수 있음.

4. **경쟁 우위 지속 가능성**
   AI-Social Studio는 이미 거의 동일한 기능을 개발 중이고, Meta API 정책상 "완전 자동 발행"이 실현 불가능하면 경쟁사 대비 실질적 차별점이 "가격"밖에 남지 않음.

## 가장 위험한 가정
**인스타그램(Meta)이 타사 앱의 '매일 완전 자동화된 콘텐츠(특히 이미지 및 릴스) 생성 및 발행'을 API를 통해 허용하고, 그 정책을 안정적으로 유지할 것이다.**

## 리스크 맵
| 리스크 | 심각도 | 대응 방안 |
|--------|--------|---------|
| Meta API 정책 변경 또는 제한 | 높음 | 하이브리드 발행 모델 준비 (자동 생성 + 사용자 승인) |
| AI 콘텐츠 품질 및 관련성 부족 | 높음 | 고도화된 프롬프트 엔지니어링 + 사용자 피드백 루프 |
| AI 콘텐츠 생성 비용 과다 | 높음 | API 비용 효율화, 유닛 이코노믹스 재검토 |
| 소상공인 실제 수요 부족 | 중간 | 30일 무료 체험 데이터 집중 수집 |
| 경쟁사 추격 및 시장 포화 | 중간 | 빠른 기능 확장, 시장 선점 효과 활용 |

### [지훈] PRD — Claude

**MVP 범위**: 카페 업종 단일, 자동 생성 + 원탭 발행 모델 (Meta API PoC 결과 반영)

#### Must Have
1. **업종 온보딩**: 업종(카페 고정), 가게 이름, 분위기 키워드(3개 선택), 대표 메뉴 입력
2. **일일 콘텐츠 자동 생성**: 매일 오전 8시, AI가 캡션(본문 + 해시태그 10개) + 정방형 이미지(1:1) 1세트 자동 생성
3. **콘텐츠 미리보기 + 수정**: 생성된 캡션/이미지를 발행 전에 확인하고 텍스트 직접 수정 가능
4. **원탭 발행**: "인스타에 올리기" 버튼 1번으로 Instagram Graph API를 통해 발행 (사용자 승인 트리거 방식)
5. **발행 예약**: 원하는 시간으로 예약 발행 설정 가능
6. **발행 히스토리**: 지난 30일 발행된 콘텐츠 목록 및 인스타 좋아요/도달 수 기본 조회
7. **구독 결제**: 월 9,900원 카드 자동결제, 첫 30일 무료 체험

#### Must NOT (범위 외)
- ❌ 완전 자동 발행 (사용자 승인 없는 자동 포스팅) — Meta API 정책상 불가
- ❌ 릴스(Reels) 생성 및 발행
- ❌ DM 자동화
- ❌ 카페 외 업종 (네일샵, 식당 등) — PMF 확인 후 확장
- ❌ 멀티 계정 관리

#### User Flow
```
가입 → 온보딩(가게 정보 입력) → Instagram OAuth 연동
→ 첫 콘텐츠 생성 → 미리보기/수정 → 원탭 발행
→ 매일 오전 8시 알림 → 확인/수정 → 발행 반복
```
