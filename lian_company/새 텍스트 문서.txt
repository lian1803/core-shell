맞아, 내가 방금 말한 Zapier/n8n 위주 얘기는 거의 23~24년식 “옛날 에이전시/노코드” 마인드고, 26년 2–3월 트렌드는 결이 많이맞아, 내가 직전에 말한 “Zapier + RPA + LLM 살짝 얹기”는 24~25년식이고, 26년 2–3월 트렌드는 “네이티브 컴퓨터 유즈 + 멀티에이전트 오케스트레이션” 쪽으로 한 단계 더 올라와 있어.

26년 초 기준 진짜 메인 흐름
단일 툴/앱 자동화가 아니라, OS·브라우저 전체를 직접 쓰는 에이전트가 중심이 되고 있음 (GPT‑5.4 Computer Use, Claude Computer Use 같은).

기업 쪽은 “에이전트 몇 개 띄워본다” 수준이 아니라,

명확한 역할 나눔이 있는 멀티에이전트(플래너/실행/검증/정책/도메인 스페셜리스트) +

그 위를 조율하는 오케스트레이션 레이어로 가는 중.

최신 플로우를 압축하면
요즘 기사/리포트에서 공통으로 나오는 “2026 스타일” 패턴을 네 케이스에 맞게 번역하면:

목표는 “태스크”가 아니라 아웃컴으로 정의

“페이지 만들어줘”가 아니라 “이 니치에서 리드 50명 확보하는 랜딩 완성 + 퍼포먼스 데이터까지 뽑아라” 식으로 책임 단위.

멀티에이전트 구조 (마이크로서비스 느낌)

플래너: 전체 플로우 설계, 어떤 툴/경로 쓸지 결정.

실행 에이전트: 리서치, 카피, 디자인 스펙, 코드, 배포, 마케팅 등 역할별로 분리.

검증/정책 에이전트: 결과 QA, 안전/규정 체크, 메트릭 모니터링.

네이티브 컴퓨터 유즈 레이어

퍼플렉시티, 웹클로드, 러버블 자체를 “API 호출”이 아니라,

컴퓨터 유즈 에이전트가 직접 브라우저/앱을 만지는 식으로 묶음.

그래서 n8n이나 Zapier 같은 중간 오케스트레이터 없이도,
“에이전트가 크롬 열고 → Perplexity 켜고 → 결과 복사 → Claude 웹에 붙여넣고 → Lovable 열어 구조 잡는” 게 가능해짐.

오케스트레이션 = 코어 인프라

구글/딜로이트/클라우드 업체 리포트 전부 “에이전트 오케스트레이션이 26년 인플렉션 포인트”라고 박고 있어.

요지는 “한 에이전트 잘 만드는 것”보다
“여러 에이전트를 역할·권한·메트릭·에러 처리까지 포함해서 운영 체계로 올리는 것”이 핵심이라는 거.

너가 하고 싶은 건 딱 이 최신 모델이랑 맞물려:

사람 리안이 하던 “퍼플렉시티 → 웹클로드 → 러버블” 전체를,

상단의 플래너 + 중간의 역할별 에이전트 + 하단의 Computer Use 에이전트로 흉내 내는 리안 버전 26년식 오케스트레이터를 만드는 거거든.

그래서 다음 스텝으로,
“리안 컴퍼니 기준 26년형 구조”를 아주 러프하게라도 3–4개 에이전트 + 1 컴퓨터유즈 레이어 수준으로 그려볼까, 아니면 먼저 “어떤 작업(예: 랜딩1종, 유튜브 스크립트, 인스타 캠페인)”에만 이 구조를 적용할지부터 좁혀볼까?

그니까 그게 어떤 툴이냐고? 클로드코드야? 아니면 뭐야
1 단계 완료
지금 말한 “컴퓨터 제어 기능”은 클로드코드가 아니라, 별도의 Claude Computer Use 기능이야.

뭐에 붙어 있는 기능인지
이름: Claude Computer Use.

제공 방식:

Anthropic API에서 Computer Use 지원 모델(예: Claude 3.5 Sonnet + Computer Use)로 호출하거나,

별도 런타임/클라이언트(데스크톱/브라우저 에이전트)가 설치된 환경에서, 그 런타임이 스크린샷·마우스·키보드 이벤트를 대신 실행해 주는 구조.

클로드코드(IDE 안에서 쓰는 Claude)는 이걸 내장해서 쓸 수도 있지만, 기능 자체의 이름/정체는 “Computer Use API/모델” 계열이라고 보면 돼.

즉, “어떤 툴이냐”로 딱 말하면:
클로드코드 같은 앱 이름이 아니라, Claude용 컴퓨터 제어 전용 기능/모델(Computer Use) 이고, 이걸 IDE(클로드코드), 데스크톱 에이전트, 너가 짜는 파이썬 오케스트레이터 어디에든 붙여서 쓰는 거라고 이해하면 돼.

지금 리안은 이걸 LIANCP 파이프라인에 API로 붙여서 네가 쓰는 컴퓨터를 조작하게 만들고 싶은 거야, 아니면 우선 클로드코드 안에서 이 기능을 활용하는 사용법부터 알고 싶은 거야?

어디서 어떻게 하는데
1 단계 완료
지금 말한 “어디서 어떻게 하냐”를 딱 끊어서 말하면:

1. 어디서 쓰는 거냐
이 기능은 Anthropic의 Claude Computer Use고,
네 PC에 띄운 “가상 데스크톱”을 Claude가 조작하는 구조야.

OpenAI Operator처럼 웹에서 바로 쓰는 게 아니라,
네가 도커 컨테이너(리눅스 데스크톱) 하나 띄워주고, Claude가 그 안에서 마우스·키보드를 움직이는 형식.

2. 어떻게 시작하냐 (개념)
Anthropic API 키 발급

Docker 설치

터미널에서 Anthropic가 제공하는 Computer Use 데모 컨테이너 실행:

bash
export ANTHROPIC_API_KEY=네키
docker run \
  -e ANTHROPIC_API_KEY=$ANTHROPIC_API_KEY \
  -p 8080:8080 \
  -p 5900:5900 \
  ghcr.io/anthropics/anthropic-quickstarts:computer-use-demo-latest
브라우저에서 http://localhost:8080 접속하면
왼쪽에 Claude 채팅, 오른쪽에 “가상 데스크톱 화면”이 같이 보임.

거기서 Claude한테
“파이어폭스 열고 구글 들어가서 XXX 검색해” 같은 식으로 시키면,
진짜로 그 오른쪽 데스크톱에서 마우스 움직이고 타이핑함.

3. 중요한 포인트
이건 “클로드코드 안 옵션 켜는 것”이 아니라,
별도 컴퓨터-유즈 환경(도커 기반 가상 데스크톱)을 하나 깔고,
Claude를 그 환경에 붙여서 쓰는 느낌이야.

나중에 네 LIANCP 오케스트레이터에서 API로 부를 때도,
결국 저 도커/VM 같은 “에이전트용 데스크톱”을 하나 띄워놓고,
거기서 퍼플렉시티/웹클로드/러버블을 실제로 여는 구조가 기본 패턴.

지금 단계에서 궁금한 건, 이거를 “일단 데모 수준으로 내 로컬에서 한번 경험해보고 싶다”냐, 아니면 “바로 LIANCP 파이프라인이 저 도커 환경이랑 붙는 구조 설계를 해보자”냐?
