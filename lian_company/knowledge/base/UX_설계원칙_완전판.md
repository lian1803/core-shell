# UX/UI & 사용자 중심 설계 강의 핵심 체크리스트

## 1. UCD (사용자 중심 설계) 정의와 원칙
강의 전문에는 사용자 중심 설계(UCD)에 대한 명확한 정의나 특정 원칙 목록이 제시되어 있지는 않으나, 강의 전체가 사용자 중심 철학을 기반으로 하고 있으며, 그 정신이 아래 요소들에 반영되어 있습니다.

*   **UCD 철학 반영 요소:**
    *   [X] **인간에 대한 깊은 이해:** 결국 사용자는 사람이라는 인식을 바탕으로 페르소나 기반 사고와 공감 능력을 핵심으로 강조합니다.
    *   [X] **사용자 문제 해결:** 사용자의 고통(Pain Point)을 발견하고 그것을 해결하는 것을 목표로 합니다.
    *   [X] **Early Adopter 집중:** 문제를 가장 심하게 겪는 초기 사용자의 피드백을 통해 제품의 방향을 결정합니다.
    *   [X] **가설 검증:** 사용자의 니즈와 문제를 가설로 설정하고 빠르게 검증하는 과정을 중요하게 생각합니다.
    *   [X] **쉬운 사용성:** "50대 부모님도 쉽게 쓸 수 있도록..."과 같이 다양한 사용자가 쉽게 접근하고 사용할 수 있도록 고려합니다.

## 2. 린 스타트업 4가지 가설 (고객/문제/솔루션/비즈니스)
린 스타트업의 핵심 가설을 세우고 빠르게 검증하는 방법론입니다.

*   **핵심 원칙:** 가설은 "정답"이 아니라 "검증할 대상"이며, 빠르게 세우고 빠르게 틀리는 것이 중요합니다.

### 2.1. 고객 가설 (Customer Hypothesis)
*   [X] **정의:** "누가 이 문제를 겪고 있는가?"를 구체적으로 정의하는 것.
*   [X] **Early Adopter의 중요성:**
    *   모든 사람을 타겟하면 아무도 만족시킬 수 없음.
    *   문제를 가장 심하게 겪어 불완전한 솔루션도 기꺼이 사용할 사람.
    *   이들의 피드백이 제품의 방향을 결정함.
*   [X] **Key Point:** "20대 여성" 같은 인구통계학적 정보만으로는 부족하며, **행동과 상황**을 함께 정의해야 함.
*   [X] **템플릿:** `[누가] [어떤 상황에서] [어떤 불편함]을 겪고 있다.`
*   [X] **자가 점검 체크리스트:**
    *   [X] 충분히 구체적인가? (좁은 범위)
    *   [X] 실제로 찾을 수 있는 사람인가?
    *   [X] 직접 인터뷰할 수 있는가?
    *   [X] 이 사람이 돈을 낼 만큼 문제가 심각한가?

### 2.2. 문제 가설 (Problem Hypothesis)
*   [X] **정의:** 고객이 겪고 있는 **Top 3 문제**를 정의하는 것.
*   [X] **핵심:** "문제를 이해하지 못하면 올바른 솔루션을 만들 수 없다."
*   [X] **기존 대안(Existing Alternatives) 파악의 중요성:**
    *   고객이 현재 이 문제를 어떻게 해결하고 있는지 파악해야 함.
    *   기존 대안이 없다면, 문제가 충분히 심각하지 않을 수 있음.
*   [X] **AI 활용 팁:** AI는 "가설 생성"을 도와줄 뿐이며, 제시된 문제를 반드시 실제 사용자에게 확인(인터뷰)해야 함.

### 2.3. 솔루션 가설 (Solution Hypothesis)
*   [X] **정의:** 각 문제에 대한 해결책을 정의하는 것.
*   [X] **핵심:** **MVP(Minimum Viable Product)**로 가장 작게 시작해야 함 (완벽한 제품보다 핵심 가치 하나만 검증할 수 있는 최소 기능 제품부터).
*   [X] **순서 (Demo, Sell, Build):** 데모로 보여주고 판매 가능성을 확인한 다음에 만들어야 함.
*   [X] **MVP 범위 정의:** 1주 안에 만들 수 있는 수준으로 최소 범위 정의.

### 2.4. 비즈니스 가설 (Business Hypothesis)
*   [X] **Revenue Streams (수익 모델):** 어떻게 돈을 벌 것인가? (구독, 수수료, 광고, 프리미엄, 라이선스 등)
*   [X] **Cost Structure (핵심 비용):** 무엇이 핵심 비용인가? (개발비, 서버비, 인건비, 마케팅비, 물류비 등)
*   [X] **Key Metrics (성공 지표):** 어떤 지표로 성공을 측정할 것인가? (MAU, 전환율, 리텐션, LTV, CAC 등)
*   [X] **Unfair Advantage (경쟁 우위):** 쉽게 복제할 수 없는 경쟁 우위는 무엇인가? (독점 데이터, 네트워크 효과, 브랜드, 특허, 커뮤니티 등)

## 3. User Journey Map 작성 방법 (단계별)
사용자가 서비스를 이용하는 전체 과정을 시각화하여 고통(Pain Point)을 발견하고 해결하는 도구.

*   [X] **정의:** 사용자가 서비스를 이용하는 전체 과정을 시각화한 도구.
*   [X] **핵심 구성요소:**
    *   [X] 페르소나 (누가 사용하는가)
    *   [X] 단계 & 행동 (어떤 순서로 무엇을 하는가)
    *   [X] 감정곡선 (각 단계에서 어떤 기분인가)
    *   [X] 페인포인트 (어디서 불편함을 느끼는가)
*   [X] **AI로 기존 서비스 유저저니 분석하기:**
    *   [X] **프롬프트:** `당신은 10년차 UX 디자이너입니다. [서비스명] 의 사용자 여정을 분석해주세요. 다음 형식으로 정리해주세요: | 단계 | 행동 | 터치포인트 | 감정 | 페인포인트 | 개선 기회 | 단계는 다음을 포함해주세요: 인지 고려 온보딩 첫 사용 정기 사용 추천/이탈 각 단계마다 구체적인 사용자 행동과 감정을 포함해주세요.`
    *   [X] **Pro Tip:** 앱 주요 화면 스크린샷 3-5개를 함께 첨부하면 훨씬 정확한 분석 가능. (`ChatGPT, Claude 모두 이미지 분석을 지원`)
*   [X] **내 서비스 유저저니 직접 작성하기 (6단계):**
    *   [X] 인지 (Awareness)
    *   [X] 고려 (Consideration)
    *   [X] 온보딩 (Onboarding)
    *   [X] 첫 사용 (First Use)
    *   [X] 정기 사용 (Regular Use)
    *   [X] 추천 (Advocacy)
*   [X] **각 단계별 채울 항목:**
    *   [X] 행동: 사용자가 무엇을 하는가
    *   [X] 생각/기대: 무엇을 기대하는가
    *   [X] 감정: 어떤 기분인가
    *   [X] 터치포인트: 어디서 접하는가
    *   [X] 페인포인트: 어디서 불편한가
    *   [X] 기회요소: 어떻게 개선할 수 있는가
*   [X] **AI로 유저저니 피드백 받기:**
    *   [X] 작성한 유저저니 테이블을 AI에 붙여넣고 피드백 요청.
    *   [X] **피드백 관점:**
        1.  빠진 단계나 터치포인트가 있는지
        2.  각 단계의 페인포인트가 충분히 구체적인지
        3.  추가로 고려해야 할 페인포인트
        4.  개선 효과가 가장 클 Top 3 지점
        5.  각 Top 3 지점에 대한 구체적 개선 방안
    *   [X] **AI 활용 워크플로우:** 직접 작성 → AI 피드백 요청 → 피드백 반영 수정 → 수정본 AI 검증 요청.

## 4. 좋은 디자인 4원칙 (명확성/일관성/피드백/단순함)
강의 전문에서 명시적으로 언급된 "반드시 지켜야 할 디자인 원칙"은 3가지(접근성, 일관성, 피드백)이며, 명확성과 단순함은 강의 전반에 걸쳐 강조되는 핵심 가치로 볼 수 있습니다.

### 4.1. 접근성 (Accessibility)
*   [X] **정의:** 누구나 사용할 수 있어야 함.
*   [X] **체크리스트:**
    *   [X] 글자 크기가 충분히 큰가?
    *   [X] 배경과 글자의 명도 대비가 확실한가?
    *   [X] 색맹 사용자도 구분 가능한가?

### 4.2. 일관성 (Consistency)
*   [X] **정의:** 학습 비용을 줄여줌.
*   [X] **체크리스트:**
    *   [X] 버튼 모양이 통일되어 있는가?
    *   [X] 같은 기능은 같은 위치에 있는가?
    *   [X] 용어가 통일되어 있는가?
*   [X] **활용:** 스타일 가이드(Style Guide)를 만들어 일관성을 유지할 수 있음.

### 4.3. 피드백 (Feedback)
*   [X] **정의:** 내 행동에 시스템이 반응해야 함.
*   [X] **체크리스트:**
    *   [X] 버튼을 눌렀을 때 눌린 효과가 있는가?
    *   [X] 로딩 중에 로딩 표시가 나오는가?
    *   [X] 에러 발생 시 이유를 알려주는가?
*   [X] **좋은 피드백 예시:**
    *   입력 즉시 유효성 검사 (실시간 피드백).
    *   로딩 시 스켈레톤(Skeleton) UI나 스피너 표시.
    *   에러 발생 시 친절한 설명과 해결책 제시.

### 4.4. 명확성 (Clarity) & 단순함 (Simplicity)
강의에서 직접적인 원칙으로 명시되지는 않았으나, UI/UX 디자인의 핵심 가치로 강조됩니다.
*   [X] **명확성 강조 요소:**
    *   [X] **좋은 UI vs 나쁜 UI:** 에러 메시지 등에서 친절한 설명과 해결책 제시를 통해 명확한 정보 전달 강조 (Slide 29).
    *   [X] **행동 명확성:** 사용자가 어디를 눌러야 할지 바로 알 수 있도록 하는 것 (Slide 46).
    *   [X] **질문 능력:** AI에게 명확하고 구체적으로 요청하는 힘 (Slide 19).
*   [X] **단순함 강조 요소:**
    *   [X] **린 스타트업:** MVP로 가장 작게 시작 (Slide 7).
    *   [X] **로그인 화면:** "간결함이 생명" 강조 (Slide 34).
    *   [X] **모바일 퍼스트:** "핵심에 집중"을 통해 불필요한 요소 제거 (Slide 30).
    *   [X] **키워드:** "Minimal(미니멀)" 스타일 키워드 활용 (Slide 21).
    *   [X] **대시보드:** "정보 과다"를 피하고 "적절한 밀도"와 "충분한 여백"을 강조 (Slide 36).

## 5. 공감→정의→아이디어→프로토타입→테스트 전 과정 (Design Thinking Process)
강의는 Design Thinking의 5단계를 명시적으로 제시하지 않지만, 각 단계에 해당하는 활동들을 Lean Startup과 AI 활용 프로세스에 녹여 설명합니다.

*   [X] **공감 (Empathize):**
    *   [X] 고객 가설: "누가 이 문제를 겪고 있는가?" 정의 (Slide 3).
    *   [X] 인간에 대한 깊은 이해 및 공감 능력 (Slide 19).
    *   [X] 사용자 여정 맵을 통해 사용자의 감정곡선과 페인포인트 시각화 (Slide 13, 17).
*   [X] **정의 (Define):**
    *   [X] 문제 가설: 고객이 겪는 Top 3 문제 정의 (Slide 5).
    *   [X] 사용자 여정 맵에서 가장 큰 페인포인트에 집중 (Slide 17).
*   [X] **아이디어 (Ideate):**
    *   [X] 솔루션 가설: 각 문제에 대한 해결책 정의 (Slide 7).
    *   [X] AI를 통한 솔루션 아이디어 확장 (Slide 8).
    *   [X] AI의 조합, 확장 능력 활용하여 다양한 디자인 시안 생성 (Slide 18).
*   [X] **프로토타입 (Prototype):**
    *   [X] 솔루션 가설: MVP(Minimum Viable Product)로 가장 작게 시작 (Slide 7).
    *   [X] Stitch: 빠르고 쉬운 웹사이트 초안/랜딩페이지 프로토타이핑 (Slide 24).
    *   [X] Figma: 업계 표준 디자인 툴에서 AI 플러그인을 활용한 와이어프레임 생성 (Slide 25).
    *   [X] Readdy AI: 구조적인 웹사이트를 빠르게 생성 및 커스터마이징 (Slide 26).
    *   [X] v0.dev: 자연어 설명으로 React + Tailwind CSS 코드 자동 생성 (Slide 27).
*   [X] **테스트 (Test):**
    *   [X] 린 스타트업 핵심: Plan A를 빠르게 세우고, 빠르게 검증하고, 빠르게 수정 (Slide 3).
    *   [X] 문제 가설 검증: AI가 제시한 문제를 실제 사용자에게 확인(인터뷰) (Slide 6).
    *   [X] 솔루션 검증 방법 정의 (Slide 8).
    *   [X] AI로 유저저니 피드백 받기 (Slide 16).
    *   [X] 프롬프트 반복 개선(Iteration)을 통한 테스트 및 수정 (Slide 39).
    *   [X] AI에게 디자인 피드백 프롬프트 요청 (Slide 40).
    *   [X] 피어 리뷰 (Peer Review)를 통한 사용자/디자인 피드백 (Slide 46).
    *   [X] A/B 테스트와 AI 활용 (Slide 56).
    *   [X] AI 페르소나 사용성 테스트 (Slide 57).
    *   [X] 사용자 피드백 받기 (주변 사람에게 보여주고 질문) (Slide 59).

### 6.2. Figma (업계 표준 디자인 툴 + AI 플러그인)

## 7. UI/UX 프롬프트 작성법과 예시
AI 시대에 필요한 '좋은 질문' 능력과 '키워드를 다루는 힘'을 통해 원하는 디자인을 얻는 방법.

*   [X] **AI 시대의 핵심 능력 (Slide 19):**
    1.  **질문 능력:** 명확하고 구체적인 요청 (역할 부여, 맥락 제공, 구체적 요구사항, 제약 조건, 결과물 형태).
    2.  **키워드를 다루는 힘:** 스타일, 분위기, 트렌드를 키워드로 조합하여 원하는 결과 도출.
    3.  **실행력과 실험정신:** 빠르게 여러 버전 만들고 비교 (완벽보다 속도).
    4.  **인간에 대한 깊은 이해:** 페르소나 기반 사고와 공감 능력.
*   [X] **좋은 질문의 구조 (5가지 핵심 요소) (Slide 20):**
    *   [X] **역할 (Role):** "너는 10년차 UI 디자이너야."
    *   [X] **맥락 (Context):** "20대 여성을 위한 쇼핑몰이야."
    *   [X] **요구 (Task):** "메인 화면 레이아웃을 잡아줘."
    *   [X] **제약 (Constraints):** "복잡하지 않게, 모바일 우선으로."
    *   [X] **형식 (Format):** "HTML/CSS 코드로 줘."
*   [X] **키워드 조합의 힘 (공식) (Slide 21):**
    *   [X] `스타일 + 분위기 + 컬러 + 레이아웃 = 완벽한 결과`
    *   [X] **스타일:** Minimal, Material, Glassmorphism, Neumorphism, Flat 등.
    *   [X] **분위기:** Professional, Playful, Luxurious, Trustworthy, Energetic 등.
    *   [X] **컬러:** Pastel, Vivid, Dark Mode, Monochrome, Gradient 등.
    *   [X] **레이아웃:** Grid System, Card UI, Split Screen, Full Screen, Sidebar 등.
*   [X] **AI에게 UI 디자인 요청 실습 프롬프트 (Slide 22):**
    `당신은 전문 UI/UX 디자이너입니다. [서비스명] 이라는 서비스를 만들고 있습니다. 타겟 유저는 [타겟] 입니다. 메인 화면의 UI 디자인을 제안해주세요. - 스타일: [키워드 1] - 분위기: [키워드 2] - 주요 컬러: [키워드 3] - 레이아웃: [키워드 4] 구체적인 섹션 구성과 각 섹션에 들어갈 내용을 텍스트로 상세히 묘사해주세요.`
    *   **Tip:** AI가 그려주는 이미지가 아니라, "텍스트 묘사"를 먼저 받는 것이 좋음.

## 8. 디자인 피드백 루프 방법
AI와 사용자, 동료로부터 디자인에 대한 피드백을 받고 개선하는 과정입니다.

*   [X] **프롬프트 반복 개선 (Iteration) (Slide 39):**
    *   [X] AI와의 대화는 한 번으로 끝나지 않음. 원하는 결과가 나올 때까지 계속 다듬어감.
    *   [X] **단계:** 초안 생성 → 문제 발견 → 구체적 수정.
    *   [X] **대화 예시:** "로그인 화면 만들어줘." → (AI 결과) → "너무 복잡해. 미니멀하게 바꿔줘. 배경색 흰색, 그림자 줄여줘." → (AI 수정) → "좋아. 로그인 버튼 색상 #6366f1, 모서리 더 둥글게(20px) 해줘."
*   [X] **디자인 피드백 프롬프트 (AI 활용) (Slide 40):**
    *   [X] **역할 부여:** `Act as a Senior UX Researcher.`
    *   [X] **검토 기준:** `Review the attached UI design (or code) based on: 1. Usability Heuristics (사용성) 2. Visual Hierarchy (시각적 위계) 3. Accessibility (접근성)`
    *   [X] **요청 내용:** `Point out 3 bad UX patterns and suggest improvements.`
    *   [X] **스크린샷 활용:** Claude나 ChatGPT(Vision)에 스크린샷 이미지를 첨부하여 요청하면 정확한 리뷰 가능.
*   [X] **피어 리뷰 (Peer Review) (Slide 46):**
    *   [X] 동료에게 결과물 공유 및 피드백 받기.
    *   [X] **피드백 관점:**
        *   [X] **첫인상 (First Impression):** 무슨 서비스인지, 분위기는 어떤지.
        *   [X] **행동 명확성 (Clarity):** 어디를 눌러야 할지 바로 알 수 있는지, 헤매지 않는지.
        *   [X] **일관성 (Consistency):** 여러 화면이 하나의 앱처럼 느껴지는지.
    *   [X] AI에게도 피드백을 요청하여 객관적 시선 활용.
*   [X] **AI 페르소나 사용성 테스트 (Slide 57):**
    *   [X] AI를 가상의 사용자로 빙의시켜 피드백 받기 (예: "55세 IT 기기에 익숙하지 않은 사람").
    *   [X] **피드백 요청 내용:** 1) 앱이 무엇을 하는지, 2) 무엇을 먼저 누를지, 3) 혼란스러운 점은 없는지, 4) 개인 정보를 신뢰할 수 있는지 등.
    *   [X] **활용법:** 타겟 페르소나와 정반대되는 페르소나에게 각각 피드백을 요청하여 비교 분석.
*   [X] **사용자 피드백 받기 (강의 후 액션 아이템) (Slide 59):**
    *   [X] 주변 사람 3명에게 보여주고 "이게 무슨 서비스 같아?"라고 물어보기.
